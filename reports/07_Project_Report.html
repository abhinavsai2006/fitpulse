1. Abstract

FitPulse is a modern fitness tracking web application designed to provide users with a unified platform for workouts, activity insights, goal planning, and profile management. The project evolved from a UI-focused implementation to a cloud-integrated system using Firebase Authentication and Firestore for secure account and data handling.

2. Problem Statement

Users often manage fitness data across disconnected tools, creating inconsistent tracking and low engagement. FitPulse addresses this by offering a single responsive application where authentication, progress monitoring, and account personalization are consolidated into one workflow.

3. Objectives and Scope

3.1 Objectives

  • Deliver a clean and responsive fitness platform with modular frontend design.
  • Implement robust authentication including verification and recovery workflows.
  • Persist user profile and onboarding information securely in cloud storage.
  • Achieve deployment readiness with reproducible environment configuration.

3.2 Scope

The delivered scope includes landing/auth flows, onboarding, dashboard, workouts, activity, nutrition, analytics, goals, profile, and settings modules with Firebase-backed account management and Firestore persistence for user-centric data.

4. Development Methodology

The execution followed an iterative implementation cycle: module development, integration, validation, and correction. High-impact issues were prioritized first, especially authentication stability, secure configuration, and production deployment consistency.

6.1 Data and control flow

Authentication events are handled through AppContext, which synchronizes session state with Firebase and updates UI routing. Firestore acts as the source of truth for user profile and onboarding details. This design keeps business logic centralized and minimizes state drift between pages.
