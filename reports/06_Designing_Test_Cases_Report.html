1. Introduction

This report defines comprehensive test cases for the latest FitPulse baseline, including Firebase Authentication, Firestore profile persistence, environment-variable based configuration, and production deployment behavior.

1.1 Test objectives

  • Validate all critical user journeys from authentication to dashboard usage.
  • Verify verified-email policy, password reset, and Google sign-in fallback behavior.
  • Ensure Firestore data writes and reads obey owner-only security rules.
  • Confirm responsive behavior and full-screen mobile experience remain stable.
  • Confirm release readiness through successful build and deployment checks.

2. Test Strategy and Methodology

2.1 Test levels

  • Component-level checks for key reusable widgets and auth UI interactions.
  • Integration checks for AppContext actions and Firebase operations.
  • End-to-end manual scenarios from registration to onboarding and dashboard.
  • Deployment-level sanity checks on live production endpoint.
