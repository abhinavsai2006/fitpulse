# FitPulse Final Print/PDF Export Checklist

## 1) Pre-Export Readiness
- [ ] Confirm all 3 report source files are final and saved:
  - 05_Implementation_Coding_Report.html
  - 06_Designing_Test_Cases_Report.html
  - 07_Project_Report.html
- [ ] Confirm links are correct and reachable:
  - Live: https://fitpulse-swe.vercel.app/
  - GitHub: https://github.com/abhinavsai2006/fitpulse
- [ ] Confirm all dates, names, registration numbers, and course details are accurate.
- [ ] Confirm report language and formatting are final (no TODOs, placeholders, or draft notes).

## 2) Browser Print Setup (for each report)
- [ ] Open the HTML report in browser.
- [ ] Press Ctrl+P (Print).
- [ ] Destination: Save as PDF.
- [ ] Paper size: A4.
- [ ] Orientation: Portrait.
- [ ] Margins: Default (or None only if content alignment requires it).
- [ ] Scale: 100% (or Fit to page only if clipping is observed).
- [ ] Enable Background graphics (important for styles and visual consistency).
- [ ] Disable Headers and footers (to avoid browser URL/date overlays).
- [ ] Verify page breaks are clean and no section is cut between pages.

## 3) Export File Naming (exact)
- [ ] Export with final names:
  - 05_Implementation_Coding_Report.pdf
  - 06_Designing_Test_Cases_Report.pdf
  - 07_Project_Report.pdf

## 4) Visual QA After Export
- [ ] Cover page is centered and complete.
- [ ] Table of contents appears correctly.
- [ ] Mermaid diagrams are rendered and readable.
- [ ] Tables are not clipped and row text is fully visible.
- [ ] Page footer is visible and consistent.
- [ ] No blank pages added unintentionally.
- [ ] No missing fonts, broken symbols, or layout shifts.

## 5) Content QA Before Submission
- [ ] Implementation report includes latest Firebase/auth/deployment updates.
- [ ] Test-case report includes updated auth/data/deployment validation coverage.
- [ ] Project report reflects final architecture, security, and operations details.
- [ ] Live and GitHub links are included in all required sections.
- [ ] No "Team Contribution Summary" section is present.

## 6) Submission Bundle Preparation
- [ ] Create a final folder: FitPulse_Final_Submission.
- [ ] Place the 3 PDF files in the folder.
- [ ] Optionally include source HTML files for reference.
- [ ] Optional archive: FitPulse_Final_Submission.zip.

## 7) Final Submission Gate
- [ ] Re-open each PDF and do one last visual scan.
- [ ] Confirm file names match rubric requirement exactly.
- [ ] Confirm upload target/course portal requirements (size, format, naming).
- [ ] Submit all files.

## 8) Post-Submission Safety
- [ ] Keep local backup copy of PDFs and HTML files.
- [ ] Keep submitted ZIP/folder copy with timestamp.
- [ ] Keep latest GitHub commit hash for traceability.
